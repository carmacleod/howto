Steps for merging new changes in a Github repo back into a local fork of the repo in Orion

Starting from scratch with a clean, synchronized workspace, here are the steps to merge changes in a Github repo (i.e. from other people) back into your local fork of the repo in Orion.
(Assuming that you already forked the repo on github and cloned it into Orion).

Start with a clean workspace

  • Navigate to the Orion Git page
  • Make sure that the Repository: dropdown points to your clone of the github repo (i.e. the fork).
  • Refresh your browser page (F5) and make sure there are no local changes.
  • Using the Reference: dropdown, checkout origin/master. You should see Reference: master => origin/master.
    i.e. the "Remote" is origin and the "Local master branch" is connected to the "Remote master branch".

One time only: Create a new Remote

  • Use the Reference: dropdown to create a New Remote.
  • The remote name should be recognizable as "the original repo".
  • Use the URL for the github repo that you originally forked from.

Select the Remote

  • Use the Reference: dropdown to select the remote.
  • You should see Reference: master => theoriginal/master.

Fetch and Merge from the Remote

  • Click Fetch to see any Incoming changes from the original remote.
  • Then click the Merge button to merge those changes into your local master branch.

Push to your Remote

  • Use the Reference: dropdown to go back to the remote for your fork (origin).
  • Click the Push button to push the changes out to your fork on github.
