Contributing to a GitHub repository using Orion

Here's the basic workflow (step by step details follow):

  • First time contributing to a specific repo:
    • Fork the GitHub repo
    • Clone your fork into Orion
    • Create the upstream remote
  • For each contribution to that repo:
    • Fetch from the upstream remote and merge into your fork
    • Create a new branch on your fork and check it out
    • Make your changes and commit them to the new branch
    • Push to your fork
    • Create the Pull Request on GitHub

First time contributing to a specific repo:

Fork the GitHub repo

  1. Fork the repository that you want to contribute to on GitHub.
    The fork will be associated with your GitHub ID.


  2. Copy the URL for the fork (make sure to copy the https url).

Clone your fork into Orion

  1. Drop down the Repository: combo, click the Clone Repository button, and paste in the URL for your fork.


  2. After the clone completes, it will be added to the Repository: list.

Create the upstream remote

  1. Use the Reference: combo to create a New Remote. The convention is to call it "upstream".
    Note: Use the URL for the github repo that you originally forked from.


For each contribution to the repo:

  1. Make sure there's an open Issue for your feature or bug fix on GitHub.
  2. Go to the Orion Git page. If you are already there, refresh the page (F5).

Fetch from the upstream remote and merge into your fork

  1. Select the upstream remote so that you see master => upstream/master in the Reference: combo.

  2. Use the Reference: combo or the Incoming Fetch button to fetch from the upstream remote.

    or


  3. If there are Incoming changes, click the Merge button to merge them into your local fork (origin/master).


  4. Expand local and checkout the master branch, so that you see master => origin/master in the Reference: combo.

  5. Click the Push button to push the Outgoing changes to your fork on GitHub.


Create a new branch on your fork and check it out

  1. Select the remote for your local fork so that you see master => origin/master in the Reference: combo.
  2. Use the Reference: combo to create a New Branch. The branch name should be issueX, where X is the Issue number.
    (GitHub requires a branch for each Pull Request).


  3. Find your new branch under local in the Reference: combo, and check it out by clicking the Checkout branch button.


  4. The Reference: should now be issueX => something/issueX [New branch].

Make your changes and commit them to the new branch

  1. Navigate to the Editor page.
  2. Make your changes and test them. If applicable, write unit tests.
  3. When your changes are ready to commit, navigate to, and refresh, the Git page.
  4. The Reference: should still be issueX => something/issueX [New branch].
  5. Review the changes and select the file(s) to commit.
  6. Click on more ... and select Add Change-ID to message and Add Signed-off-by to message.
  7. Make sure the Author and Committer name and email are correct.
    (You can enter them here, or on the Git Settings page).
  8. Describe the fix or feature in the commit message. Use issue #X to create a link to the issue.
  9. Click Commit to commit your changes to your issueX branch.

Clean up your workspace

  • Clear the filter in the Reference: dropdown, and check out the master branch under local.
  • The Reference: should be Reference: master => origin/master.
  • Click Sync to bring in any incoming changes.
  • You are now ready to create a new branch to work on a new bug.
